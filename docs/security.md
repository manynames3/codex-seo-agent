# Security Model

## Authentication And Authorization

- The Lambda Function URL is public, but protected at the application layer with a bearer admin token.
- The admin token is generated by `serverless/scripts/deploy-lambda.sh`, stored locally in `.local/aws-admin-token.txt`, and stored in SSM SecureString.
- The same token is stored as a Cloudflare Pages secret named `BACKEND_ADMIN_TOKEN`.
- Browser users call same-origin Cloudflare Pages Function routes, so the Lambda URL and backend token are not exposed in the UI or browser JavaScript.

## Google OAuth

- The hosted flow requires a Google OAuth Web application client.
- The client JSON is stored in SSM SecureString.
- The Search Console OAuth token JSON is stored in SSM SecureString after callback.
- Only the Search Console readonly scope is requested.

## IAM

The Lambda role can:

- Write/read/list the private reports bucket.
- Read/write the named SSM parameters used by this app.
- Write CloudWatch Logs through `AWSLambdaBasicExecutionRole`.

It does not have broad S3, IAM, DynamoDB, or account-admin access.

## Network Boundaries

- Lambda is not in a VPC. This avoids NAT Gateway cost and allows direct calls to Google/DataForSEO APIs.
- S3 public access is blocked.
- Function URL CORS currently allows all origins. The browser-facing path now goes through Cloudflare Pages, but a stricter origin allowlist would still be better for production.

## Audit Logging

- Lambda writes execution logs to CloudWatch Logs.
- AWS CloudTrail records SSM, Lambda, CloudFormation, and S3 control-plane actions if enabled on the account.
- The app does not yet persist a separate audit table for report runs.

## Known Security Gaps

- No Cognito, Cloudflare Access, or SSO.
- No WAF or custom domain.
- No per-user permissions.
- No token rotation workflow beyond redeploying/regenerating the local admin token and updating the matching Pages secret.

## Better Next Version

- Put the UI behind Cloudflare Access or Cognito.
- Restrict Function URL CORS to the Pages domain.
- Move run metadata into an append-only audit table.
- Add CloudWatch alarms for auth failures and Lambda errors.
